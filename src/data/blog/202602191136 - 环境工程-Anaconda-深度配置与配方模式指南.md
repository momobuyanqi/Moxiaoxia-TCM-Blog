# 🐍 环境工程：Anaconda 深度配置与“配方”模式指南

在 TCM 博物馆的数字化建设中，开发环境的稳定性与可迁移性是生产力的基石。今日我们正式引入了 Anaconda (Conda) 管理体系，并确立了“厨房与配方”的分离哲学。

## 1. 为什么选择 Anaconda？
对于多项目并行的社会企业实验室，环境污染是最大的敌人。Anaconda 允许我们为“一人大学”、“跨越数字鸿沟”等不同项目开辟完全隔离的虚拟空间。
- **架构适配**：针对 Mac (Apple Silicon) 的 `arm64` 优化，确保了在高性能芯片上的原生运行速度。

## 2. “厨房与配方”的哲学
为了兼顾“云端协作”与“本地效能”，我们制定了以下规范：
- **厨房 (Local ENV)**：真实的 Python 环境安装在系统主目录。它巨大且沉重，不应进入 Git 追踪。
- **配方 (environment.yml)**：每个项目文件夹下留存一份 YAML 配方文件，记录了所需的所有库。
- **意义**：配方随代码上云，确保了 momo 在任何一台设备上，都能通过一行命令重建一模一样的“厨房”。

## 3. 标准化操作流
- **创建**：`conda env create -f environment.yml`
- **激活**：`conda activate TCM-[ProjectName]`
- **更新**：当引入新库时，通过 `conda env export` 重新导出配方。

通过这种方式，我们实现了开发环境的“轻量化”与“标准化”，为未来的社会项目大规模孵化奠定了基础。
