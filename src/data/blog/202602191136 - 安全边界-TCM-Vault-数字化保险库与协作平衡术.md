# 🔒 安全边界：TCM_Vault：数字化保险库与协作平衡术

在“公开学习”与“数据隐私”之间，我们需要一道坚固的防火墙。`TCM_Vault` 的诞生，正是为了解决多端协作中敏感密钥（Token）的存储难题。

## 1. 保险库的隔离原理
我们将所有的 API 密钥、密码和私密配置统一收纳在根目录的 `TCM_Vault` 目录中。
- **封锁机制**：通过在 `.gitignore` 中设置最高的屏蔽优先级，确保整个文件夹物理降维，绝不触碰 GitHub 云端。

## 2. 协作平衡术
- **本地性**：密钥留在每一台物理设备本地，通过物理手段（如私密云盘或 U 盘）在 momo 的不同 Mac 之间同步。
- **引用逻辑**：Workshop 中的生产代码通过相对路径访问 Vault。代码（逻辑）是公开的，但燃料（密钥）是私有的。

## 3. 意义：让 AI 在安全区工作
有了 `TCM_Vault`，我们可以放心地授权 AI 助手处理复杂的自动化任务。即使 AI 误生成了包含敏感信息的文件，只要我们规定其存储在 Vault 命名空间内，就拥有了最后的“防回流”安全底线。

安全不是对协作的阻碍，而是让自由探索更有底气的保障。
